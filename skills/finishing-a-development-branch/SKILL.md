# 收尾开发分支

## 概述

通过清晰选项与所选工作流，引导开发工作收尾。

**核心原则：** 验证测试 → 呈现选项 → 执行选择 → 清理。

**开始时宣告：**「我正在使用 finishing-a-development-branch 技能来完成这项工作。」

## 流程

### 第 1 步：验证测试

**在呈现选项之前，确认测试通过：**

```bash
# 运行项目测试套件
npm test / cargo test / pytest / go test ./...
```

**若测试失败：**
```
测试失败（<N> 处）。必须先修复才能收尾：

[展示失败]

在测试通过前不能进行合并/PR。
```

停止。不要进入第 2 步。

**若测试通过：** 进入第 2 步。

### 第 2 步：确定基分支

```bash
# 尝试常见基分支
git merge-base HEAD main 2>/dev/null || git merge-base HEAD master 2>/dev/null
```

或询问：「本分支是从 main 拉的吗？」

### 第 3 步：呈现选项

**只**呈现以下 4 项：

```
实现已完成。接下来你想怎么做？

1. 在本地合并回 <基分支>
2. 推送并创建 Pull Request
3. 保留分支不动（稍后我自己处理）
4. 丢弃本次工作

选哪一项？
```

**不要附加长篇说明** — 保持选项简短。

### 第 4 步：执行选择

#### 选项 1：本地合并

```bash
# 切换到基分支
git checkout <基分支>

# 拉取最新
git pull

# 合并功能分支
git merge <功能分支>

# 在合并结果上验证测试
<测试命令>

# 若测试通过
git branch -d <功能分支>
```

然后：清理 worktree（第 5 步）

#### 选项 2：推送并创建 PR

```bash
# 推送分支
git push -u origin <功能分支>

# 创建 PR
gh pr create --title "<标题>" --body "$(cat <<'EOF'
## 摘要
<2–3 条变更要点>

## 测试计划
- [ ] <验证步骤>
EOF
)"
```

然后：清理 worktree（第 5 步）

#### 选项 3：保持不动

报告：「保留分支 <名称>。Worktree 保留在 <路径>。」

**不要**清理 worktree。

#### 选项 4：丢弃

**先确认：**
```
将永久删除：
- 分支 <名称>
- 全部提交：<提交列表>
- 位于 <路径> 的 worktree

输入 discard 确认。
```

等待完全一致的确认。

若确认：
```bash
git checkout <基分支>
git branch -D <功能分支>
```

然后：清理 worktree（第 5 步）

### 第 5 步：清理 Worktree

**对选项 1、2、4：**

检查是否在 worktree 中：
```bash
git worktree list | grep $(git branch --show-current)
```

若是：
```bash
git worktree remove <worktree-路径>
```

**对选项 3：** 保留 worktree。

## 速查

| 选项 | 合并 | 推送 | 保留 Worktree | 清理分支 |
|--------|-------|------|---------------|----------------|
| 1. 本地合并 | ✓ | - | - | ✓ |
| 2. 创建 PR | - | ✓ | ✓ | - |
| 3. 保持不动 | - | - | ✓ | - |
| 4. 丢弃 | - | - | - | ✓（强制） |

## 常见错误

**跳过测试验证**
- **问题：** 合并坏代码、创建失败 PR  
- **修复：** 呈现选项前始终验证测试  

**开放式提问**
- **问题：**「接下来怎么办？」→ 含糊  
- **修复：** 只呈现上述 4 个结构化选项  

**自动清理 worktree**
- **问题：** 在仍可能需要时删除（选项 2、3）  
- **修复：** 仅对选项 1 和 4 清理  

**丢弃无确认**
- **问题：** 误删工作  
- **修复：** 要求输入 `discard` 确认  

## 危险信号

**绝不：**
- 测试失败仍继续  
- 未在合并结果上验证测试就合并  
- 无确认就删除工作  
- 未经明确要求就 force-push  

**务必：**
- 呈现选项前先验证测试  
- 只呈现 4 个选项  
- 选项 4 须打字确认  
- 仅对选项 1 与 4 清理 worktree
