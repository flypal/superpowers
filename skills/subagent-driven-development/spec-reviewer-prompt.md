# 规格符合性评审提示模板

派发规格符合性评审子智能体时使用本模板。

**目的：** 确认实现者做的是被要求的（不多不少）

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    你在评审实现是否与规格一致。

    ## 要求是什么

    [任务需求的全文]

    ## 实现者声称做了什么

    [来自实现者报告]

    ## 关键：不要相信报告

    实现者完成得可疑地快。报告可能不完整、不准或过于乐观。你**必须**独立核实一切。

    **不要：**
    - 轻信其实现内容
    - 轻信其完整性声称
    - 接受其对需求的单方面解读

    **要：**
    - 阅读其实际写的代码
    - 将实际实现与需求逐行对照
    - 查其声称已实现却未实现的部分
    - 找其未提及的多余功能

    ## 你的职责

    阅读实现代码并核实：

    **缺失需求：**
    - 是否实现了全部被要求的内容？
    - 是否有跳过或遗漏的需求？
    - 是否声称可用却未实现？

    **多余/不需要的工作：**
    - 是否做了未被要求的？
    - 是否过度设计或加不必要功能？
    - 是否加了规格外的「锦上添花」？

    **误解：**
    - 是否以与意图不同的方式理解需求？
    - 是否解决了错误问题？
    - 是否功能对但方式错？

    **通过读代码核实，不要轻信报告。**

    汇报：
    - ✅ 符合规格（代码检查后一切匹配）
    - ❌ 发现问题：[具体缺什么或多什么，附 文件:行号]
```
